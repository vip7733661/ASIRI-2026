# API Atlas — Live Finance Edition

تطبيق ويب تقدمي (PWA) مستقل مبني فوق عينة من قسم **Finance** في مستودع `public-apis/public-apis`، مع طبقة تحقق حي دورية.

## ما الذي يعمل؟

- البحث بالاسم والوصف والوسوم.
- التصفية حسب نوع الخدمة والمصادقة والحالة الحية والمفضلة.
- فحص دوري كل ست ساعات عبر GitHub Actions.
- عرض حالة الوصول ورمز HTTP وزمن الاستجابة وآخر فحص.
- اكتشاف ترويسة CORS عندما يعيدها المزود.
- اختبار Endpoint بيانات فعلي عندما يتوفر مسار آمن بلا مفتاح؛ ويقتصر الباقي على فحص صفحة التوثيق الرسمية.
- تقييم أولي لسهولة التكامل منفصل عن الحالة الحية.
- حفظ المفضلة محليًا على الجهاز.
- تنزيل تقرير JSON يشمل الكتالوج وآخر نتائج الفحص.
- العمل دون اتصال بعد أول زيارة، مع Network-first لملف الحالة الحية.
- التثبيت على Android، والإضافة إلى الشاشة الرئيسية على iPhone.

## معنى الحالات

| الحالة | معناها |
|---|---|
| تعمل الآن | استجابة HTTP من 200 إلى 399 |
| قابلة للوصول — مقيدة | استجابة 401 أو 403 أو 405 أو 429؛ المزود موجود لكنه محمي أو محدود |
| استجابة متدهورة | Timeout أو استجابة خادم 5xx |
| غير متاحة | خطأ شبكة أو رمز HTTP آخر غير متوقع |

> الحالة الحية تقيس قابلية الوصول وزمن الاستجابة فقط. لا تثبت دقة الأسعار، أو حداثة بيانات السوق، أو اكتمال التغطية، أو حقوق إعادة الاستخدام التجاري.

## التشغيل محليًا

```bash
python3 -m http.server 4174 --directory api-intelligence
```

ثم افتح:

```text
http://localhost:4174
```

لتنفيذ الفحص الحي يدويًا من جذر المستودع:

```bash
node scripts/check-api-health.mjs \
  --catalog api-intelligence/catalog.js \
  --output api-intelligence/live-status.json
```

## الاختبارات

```bash
node --test tests/api-intelligence.test.mjs tests/api-health.test.mjs
```

## التثبيت على iPhone

1. افتح التطبيق في Safari.
2. اضغط زر المشاركة.
3. اختر **إضافة إلى الشاشة الرئيسية**.
4. اضغط **إضافة**.

## الحدود الحالية

- معظم الخدمات التي تتطلب مفتاحًا تُفحص عبر رابط التوثيق الرسمي، وليس عبر بيانات سوق فعلية.
- Free Tier وحدود الطلبات وشروط الاستخدام لا يمكن إثباتها من فحص HTTP وحده.
- بعض المزودين قد يحظرون GitHub Actions مؤقتًا؛ لذلك 403 أو 429 لا تعني أن الخدمة متوقفة.
- لا يتم إرسال مفاتيح API أو تخزينها.
- لا يوجد تداول أو اتصال بوسيط.
- Catalog Score يقيس سهولة التكامل الأولي، بينما Live Status يقيس التوفر فقط.

## المرحلة التالية

- إضافة مفاتيح تجريبية آمنة عبر GitHub Secrets للخدمات التي تسمح بذلك.
- اختبار عينات بيانات محددة وقياس الاكتمال والحداثة والدقة.
- حفظ سجل تاريخي للـUptime بدل آخر فحص فقط.
- مراجعة حقوق الاستخدام التجاري لكل مزود.
